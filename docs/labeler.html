ehr_ml.labeler¶

stride_ml.labeler is a module for defining labeling functions.

Definition Of A Labeler¶

stride_ml.labeler works on the assumption that there may be multiple or zero labels per patient.

Each label is defined by Label to consist of a day offset and a label as defined by Label. Labels can be either time to event labels, numeric labels, categorical or binary labels.

Label provides the primary interface for specifying labeling functions as code that takes in timelines and returns lists of labels.

An individual label on a particular patient at a particular time.

Construct a label with a day_index and a value. You must only provide a single value option.

class ehr_ml.labeler.FixedTimeHorizonEventLabeler¶

A fixed time horizon labeler enables you to label events that occur within a particular time horizon.

No labels are generated if the patient record is “censored” before the end of the horizon

You are required to implement two methods:

get_event_ages for specifying the times of the events get_time_horizon for specifying the length of time to use for the time horizon

class ehr_ml.labeler.InfiniteTimeHorizonEventLabeler¶

An infinite time horizon labeler enables you to label patients that have an event at any time in the future.

You are required to implement get_first_event_age for specifying the time of the first event
